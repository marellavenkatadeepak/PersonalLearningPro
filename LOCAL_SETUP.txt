MASTER PLAN - LOCAL DEVELOPMENT SETUP GUIDE

This guide will help you set up and run the Master Plan application locally on your machine.

DEPENDENCIES
------------
1. Node.js (v16 or later)
2. npm or yarn
3. Firebase project (for authentication)
4. OpenAI API key (for AI functionality)

SETUP STEPS
-----------
1. CLONE THE PROJECT
   Download the project files from Replit or copy them to your local machine.

2. INSTALL DEPENDENCIES
   Open a terminal in the project root directory and run:
   ```
   npm install
   ```
   
   Or if you use yarn:
   ```
   yarn install
   ```

3. CREATE ENVIRONMENT VARIABLES
   Create a .env file in the project root with the following variables:
   ```
   VITE_FIREBASE_API_KEY=your_firebase_api_key
   VITE_FIREBASE_PROJECT_ID=your_firebase_project_id
   VITE_FIREBASE_APP_ID=your_firebase_app_id
   OPENAI_API_KEY=your_openai_api_key
   ```

   To get Firebase credentials:
   - Go to https://console.firebase.google.com/
   - Create a new project (or use an existing one)
   - Add a web app to your project
   - Enable Google authentication
   - Copy the required credentials from the Firebase config

4. MODIFY PACKAGE.JSON
   Update the scripts section in package.json to:
   ```json
   "scripts": {
     "dev": "concurrently \"vite\" \"tsx server/index.ts\"",
     "dev:client": "vite",
     "dev:server": "tsx server/index.ts",
     "build": "vite build && esbuild server/index.ts --platform=node --packages=external --bundle --format=esm --outdir=dist",
     "start": "NODE_ENV=production node dist/index.js"
   }
   ```
   
   Also add concurrently as a dev dependency:
   ```
   npm install --save-dev concurrently
   ```

5. UPDATE VITE CONFIG
   Make sure your vite.config.ts file doesn't have any Replit-specific plugins.
   The core configuration should look like:
   ```js
   import { defineConfig } from 'vite';
   import react from '@vitejs/plugin-react';
   import path from 'path';

   export default defineConfig({
     plugins: [react()],
     resolve: {
       alias: {
         '@': path.resolve(__dirname, './client/src'),
         '@shared': path.resolve(__dirname, './shared'),
         '@assets': path.resolve(__dirname, './attached_assets')
       }
     },
     server: {
       proxy: {
         '/api': {
           target: 'http://localhost:5000',
           changeOrigin: true
         }
       }
     }
   });
   ```

6. STARTING THE APPLICATION
   Run both the frontend and backend servers:
   ```
   npm run dev
   ```

   Or start them separately:
   ```
   # Terminal 1 - Frontend
   npm run dev:client

   # Terminal 2 - Backend 
   npm run dev:server
   ```

7. ACCESS THE APPLICATION
   - Frontend: http://localhost:5173 (or the port shown in your terminal)
   - Backend: http://localhost:5000

PROJECT STRUCTURE
----------------
- client/src/ - Contains all frontend React code
  - components/ - UI components
  - contexts/ - Context providers (auth, theme, etc.)
  - hooks/ - Custom React hooks
  - lib/ - Utility functions and external service integrations
  - pages/ - Page components
  
- server/ - Contains backend Express server code
  - lib/ - Server utilities
  - routes.ts - API endpoints
  - storage.ts - Data storage implementation
  
- shared/ - Code shared between frontend and backend
  - schema.ts - Database schema definitions

REMOVING REPLIT METADATA
-----------------------
1. Delete Replit-specific files:
   - .replit
   - replit.nix
   - .config/

2. Remove any Replit-specific code from:
   - vite.config.ts (remove Replit plugins)
   - server/vite.ts (simplify to standard Vite configuration)
   - Any references to Replit domains or workflows

3. Update backend server to use a standard port:
   In server/index.ts, make sure the server listens on a specific port:
   ```
   const PORT = process.env.PORT || 5000;
   server.listen(PORT, () => {
     console.log(`Server running on port ${PORT}`);
   });
   ```

TROUBLESHOOTING
--------------
1. Port conflicts: If ports are already in use, change them in the server config and vite.config.ts
2. Firebase authentication issues: Make sure you've enabled Google authentication in Firebase
3. Environment variables: Double-check your .env file has correct values
4. Package installation errors: Try removing node_modules and package-lock.json, then reinstall

For any API key related features (OpenAI), make sure you have added the keys to your environment variables.
